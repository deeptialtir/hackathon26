What changed
  Multiline 
FormikText
 paste handling no longer treats every email as "tabular content". When the clipboard contains an email body (prose around a small inline table, or a body wrapped in a layout table — both common in Outlook/Gmail HTML), the paste now falls through to the browser's normal paste, preserving the full text exactly as it appeared in the email.
What still works
  Pasting genuine tables — from Excel, Google Sheets, web tables, etc. — continues to be auto-formatted into aligned columns inside the textarea, same as before.
How to verify
Open any email in Outlook / Gmail that contains a greeting, a short body, and at least one row/table-like item, plus a signature.
Select the entire message and copy.
In CRM, open 
Create Activity
 (or any modal with a multiline Notes field) and paste into Notes.
Expected:
 the full message text lands in the field — greeting, body, item line, and signature.
Bonus: copy a few cells from Excel/Sheets and paste into the same field — should still format as an aligned table.
